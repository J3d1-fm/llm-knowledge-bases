# Output Artifact Contract

An output is not just a final chat message. It is a reusable file or artifact that can be reopened, cited, revised, and filed back into the knowledge base.

Each output should have a title, type, status, path, summary, source coverage, and a decision state. The decision state keeps the vault from becoming a pile of unmanaged generated files.

The output format should follow the work: markdown for reports, Marp for narrative decks, tables for comparisons, charts for quantitative views, and graph maps for concept relationships.
