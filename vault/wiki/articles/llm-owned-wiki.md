# LLM-Owned Wiki

The strongest product idea is that the LLM is not only answering questions. It owns the wiki maintenance loop: reading raw material, updating pages, linking concepts, refreshing indexes, and leaving review queues.

The user should not have to manually keep every page current. The user supplies direction, source material, approvals, and questions. The agent turns that activity into durable structure.

This changes the meaning of the wiki. It is not a static note archive. It is a compiled research state that can be regenerated, audited, and improved over time.
