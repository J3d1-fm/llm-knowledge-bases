LLM Knowledge Bases Changelog

Version v0.12.0 - 2026-06-16
- Added a live Firestore Work DB context CLI so Codex can query the remote DB directly with `npm run workdb:cloud -- "<query>"`.
- Added remote-safe cloud context commands to Work DB records in Firestore and the authenticated workspace.
- Updated the authenticated detail panel to show both cloud and local Codex follow-up commands.
- Extended live Firestore validation to require cloud context commands alongside local drill-down commands.

Version v0.11.1 - 2026-06-16
- Replaced the oversized public sign-in hero with a compact top-bar Google sign-in button.
- Removed public sign-in copy that described Firebase, Firestore, allowlist, or account restrictions in the UI.
- Replaced detailed public auth/load errors with short user-facing messages while keeping diagnostics in console and validation.
- Added smoke validation to prevent implementation/security copy from returning to the sign-in screen.

Version v0.11.0 - 2026-06-16
- Changed the authenticated workspace to open on the Work DB view by default instead of the article list.
- Added cloud Work DB status diagnostics for Firestore connectivity, remote-safe privacy mode, and Codex context commands.
- Added clear fail-fast errors when Firestore `workdbContext` is empty or missing its summary document.
- Added copyable Local Codex follow-up commands inside Work DB detail records.
- Extended app smoke validation to guard the Work DB-first cloud diagnostics contract.

Version v0.10.2 - 2026-06-16
- Updated the private work graph renderer so edge line drawing and bend-dot drawing both use the same path vertex array.
- Added support for two marked bend vertices on long graph links and strengthened dimmed bend markers for zoomed-out views.
- Extended workdb graph validation and runtime debug state to require bend dots to be derived from every internal path vertex.

Version v0.10.1 - 2026-06-16
- Added a live Firestore Work DB smoke test command that validates the seeded `workdbContext` collection through Google OAuth.
- Added cloud-data render-contract checks for Work DB counts, kinds, searchability, local Codex commands, and privacy leak boundaries.

Version v0.10.0 - 2026-06-16
- Added a remote-safe Work DB context builder that derives Firestore-ready project, cluster, tag, external, and summary documents from the private local workdb.
- Extended Firestore seeding to upload the authenticated `workdbContext` collection while excluding local paths, snippets, file content, and git remotes.
- Added a Work DB tab to the authenticated workspace with project/cluster/tag search, safety-boundary notes, metrics, and local Codex follow-up commands.
- Added validation coverage for the remote context bundle privacy boundary.

Version v0.9.1 - 2026-06-16
- Updated the private work graph line renderer so long links use deterministic bend vertices instead of plain source-to-target strokes.
- Added visible bend-point markers and stronger endpoint markers so every rendered link vertex has a dot, including zoomed-out and dimmed graph states.
- Moved connection markers above the node layer so they do not disappear under later canvas drawing.
- Updated workdb graph validation to require the bend-point rendering layer.

Version v0.9.0 - 2026-06-16
- Added working local DB APIs for search, context packs, safe file preview, and localhost-only Finder reveal actions.
- Added workdb context and show CLI commands so Codex can use the private database as a fast routing/context layer before opening full source files.
- Added graph inspector actions for Context, Preview, and Reveal in Finder when tag-cloud.html is served through npm run workdb -- serve.
- Added a slow low-cost motion layer to the graph so the map subtly rotates without recalculating layout physics every frame.
- Added a generated public tag-cloud snapshot for the landing page with no labels, paths, titles, or private source names.
- Removed the old labeled hero bitmap from the landing page first viewport.
- Added scripts/build-public-snapshot.mjs and wired it into the static build.

Version v0.8.2 - 2026-06-16
- Added a connection-point layer to the memory graph so every linked node endpoint has a visible dot.
- Added visible center markers for theme cluster nodes, which were previously used as graph connection points without being drawn as dots.
- Updated workdb graph validation to require the connection-point layer in generated graph HTML.

Version v0.8.1 - 2026-06-16
- Fixed the memory graph zoom-out floor so the full clustered map can fit on screen.
- Renamed the graph fit control to Fit all and added explicit zoom out / zoom in buttons.
- Added generated graph state markers for zoom and screen bounds so local QA can verify that the graph fits inside the viewport.
- Updated workdb graph validation to catch regressions to the old zoom limits.

Version v0.8.0 - 2026-06-16
- Rebuilt the private tag-cloud output as a clustered memory graph instead of a ring-style node map.
- Added explicit theme cluster nodes and cluster membership for tags, projects, raw samples, sessions, repos, and cloud resources.

- Added graph actions that expose analysis commands for selected tags and clusters.
- Added workdb analyze-tag and analyze-cluster commands that generate markdown dossiers under outputs/global-work-kb/analysis/.
- Added a local workdb serve mode so the graph can run tag/cluster analysis through a localhost API and open the generated report.

- Added workdb graph validation to the release validation path.
- Updated docs and version metadata for the clustered memory graph release.
